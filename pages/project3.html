    How I changed that

    The website is almost built exclusively by using flexboxes, with the exception of game products preview that are displayed in CSS Grid. Flexboxes defines for me fast and simple how I want to display different elements across the page based on the different parts of the website I want to modify and change. For instance, when I make the navbar elements display in different ways even when both are displayed in flex – the main menu is controlled by justify-content, while the user-section is flex-wrapped. For the grid-part of this site, I used min-max so that the grid-columns change automatically, flexible from one screen size to another without having to change that manually through media queries.

    Why I did it that way

    Before I started using CSS Grid more frequently, I usually used flexbox to design web pages for whatever purpose and content they had. Here for now I have small design elements to implement — and this is where I could see almost purely flexbox-built website were ideal when I have smaller layout design to build and implements on, with few rows- or columns. But, then I ended up using CSS Grid anyway when the game products are previewed on the page, because it is easier for me to shape the layout of the section, in this case, in the rows and columns I had in mind with grid when there are several elements (games) to display.

    Thought after the first year I would figure out what I personally prefer and think it's most practical to use of Flexbox and CSS Grid; and so far I can tell from my view that it will depend on the type and amount of content that has to be organized across the website, or maybe this matter of fact this is more about preferences?
